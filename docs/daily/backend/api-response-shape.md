# 接口返回统一响应结构

## 适用场景

当前后端接口返回格式各不相同时，前端封装和错误处理会越来越难统一。

## 推荐做法

约定统一结构，例如：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

## 收益

- 前端更容易统一拦截器和错误提示
- 监控和日志分析更容易归类
- 联调沟通成本更低

## 注意点

- 统一结构不代表所有错误都返回 `200`
- `code`、`message` 和 `data` 的语义要在团队内保持一致
