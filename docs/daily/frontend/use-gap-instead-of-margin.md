# CSS 里优先用 Gap 而不是子项 Margin

## 适用场景

使用 Flex 或 Grid 做列表布局时，子项之间需要稳定间距。

## 推荐做法

```css
.list {
  display: flex;
  gap: 12px;
}
```

## 为什么更好

- 间距逻辑放在父容器上，更集中
- 不容易出现最后一个元素还带右边距的问题
- 列表换行时，行间距也更容易统一管理

## 注意点

- 旧项目如果兼容性要求很高，需要确认目标浏览器支持情况
- 如果是普通块级流式布局，`gap` 并不能替代所有 `margin`
