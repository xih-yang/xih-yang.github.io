# Git 撤销提交与回退选择

## 常见场景

刚提交完发现信息写错了、内容带多了，或者想撤回最近一次提交重新整理。

## 常用命令

```bash
# 撤销最近的提交，但保留修改在暂存区
git reset --soft HEAD~1

# 撤销最近的提交，保留修改在工作区
git reset --mixed HEAD~1
```

## 注意点

- `git reset --hard` 会丢掉未保存修改，日常不要乱用
- 已经推送到远程的提交，处理方式要更谨慎

## 延伸

- 只想撤销提交记录但保留改动时，优先考虑 `--soft`
- 已推送到远程的错误提交，很多场景更适合用 `git revert`
