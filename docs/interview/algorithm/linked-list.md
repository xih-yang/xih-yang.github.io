# 链表

## 高频题型

- 反转链表
- 合并两个有序链表
- 环形链表
- 删除倒数第 N 个节点

## 常见追问

- 为什么很多链表题适合用虚拟头节点
- 快慢指针分别代表什么含义

## 标准回答框架

1. 先画链表结构
2. 再说明指针变化
3. 最后补边界条件

## 题库示例

### 问题 1：反转链表的核心思路是什么

**回答要点**

- 用两个或三个指针逐步反转指向关系
- 每次要先保存下一个节点，避免链表断掉
- 重点不是代码长短，而是指针更新顺序

**常见追问**

- 为什么要先保存 `next`
- 递归写法和迭代写法怎么比较

### 问题 2：为什么很多链表题喜欢用虚拟头节点

**回答要点**

- 可以统一头节点和普通节点的处理逻辑
- 删除、插入操作时更稳定
- 能减少大量针对头节点的特殊判断

**常见追问**

- 哪些题不用虚拟头节点也能做
- 虚拟头节点会不会影响空间复杂度
