# Java 面试题

## 高频问题

- JVM、JRE、JDK 的区别
- 面向对象三大特性
- `HashMap` 原理
- 线程、线程池和并发基础

## 常见追问

- `HashMap` 为什么线程不安全
- 线程池参数怎么配置
- `synchronized` 和 `Lock` 有什么差异

## 标准回答框架

1. 先给结论
2. 再说底层机制
3. 再讲使用场景和边界

## 题库示例

### 问题 1：`HashMap` 的底层原理是什么

**回答要点**

- `HashMap` 本质是数组加链表或红黑树的结构
- 先通过 key 的 hash 定位桶位
- 冲突元素会挂在同一个桶上，元素多时会树化
- 查询效率理想情况下接近 O(1)

**常见追问**

- 为什么会出现哈希冲突
- 什么时候链表会转红黑树

### 问题 2：线程池为什么比手动创建线程更推荐

**回答要点**

- 线程池可以复用线程，减少频繁创建和销毁成本
- 可以限制并发数量，避免线程失控
- 可以通过队列和拒绝策略控制系统压力

**常见追问**

- 核心线程数和最大线程数怎么理解
- 拒绝策略分别适合什么场景

### 问题 3：`synchronized` 和 `Lock` 有什么区别

**回答要点**

- `synchronized` 是 JVM 层面的关键字，用法更直接
- `Lock` 是显式锁，功能更灵活，例如可中断、可轮询、可公平锁
- 简单同步场景可以优先用 `synchronized`

**常见追问**

- 什么是可重入锁
- 公平锁和非公平锁有什么差异
