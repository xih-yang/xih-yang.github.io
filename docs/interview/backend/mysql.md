# MySQL 面试题

## 高频问题

- 索引原理和失效场景
- 事务的四大特性
- 隔离级别与幻读
- 慢查询排查思路

## 常见追问

- 为什么最左前缀原则会影响索引命中
- 回表和覆盖索引有什么差异
- 遇到慢查询你会怎么一步步排查

## 标准回答框架

1. 先讲索引或事务原理
2. 再说查询是如何执行的
3. 最后结合排障流程来回答

## 题库示例

### 问题 1：MySQL 索引为什么能加速查询

**回答要点**

- 索引本质上是帮助数据库更快定位数据的数据结构
- InnoDB 常见索引用 B+ 树实现，适合范围查询和排序
- 有了索引后，不需要每次都全表扫描

**常见追问**

- B+ 树为什么比二叉树更适合数据库索引
- 什么情况下有索引也可能不生效

### 问题 2：什么是回表和覆盖索引

**回答要点**

- 如果索引里没有查到所有需要字段，就要回到主键或聚簇索引里再取一次数据，这叫回表
- 如果查询字段都在索引中就能拿到，就是覆盖索引
- 覆盖索引通常能减少额外 IO

**常见追问**

- 如何通过 SQL 设计尽量利用覆盖索引
- 回表一定很慢吗

### 问题 3：慢查询一般怎么排查

**回答要点**

- 先确认是不是慢 SQL，而不是锁等待或连接问题
- 再用 `EXPLAIN` 看执行计划、索引命中和扫描行数
- 最后结合业务场景决定是改 SQL、补索引还是调整表结构

**常见追问**

- `rows` 字段大说明什么
- 索引加了以后为什么还是慢
