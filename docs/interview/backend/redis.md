# Redis 面试题

## 高频问题

- 常见数据结构及使用场景
- 缓存穿透、击穿、雪崩
- 持久化机制
- 分布式锁注意点

## 常见追问

- 为什么分布式锁会有超时和误删问题
- Redis 为什么快
- 持久化和高可用方案如何取舍

## 标准回答框架

1. 先说 Redis 的核心能力
2. 再讲具体场景的风险点
3. 最后补工程上的兜底策略

## 题库示例

### 问题 1：Redis 为什么快

**回答要点**

- 基于内存读写
- 核心数据结构设计高效
- 单线程模型减少了很多锁竞争成本
- 网络模型和实现层面对高吞吐做了优化

**常见追问**

- 单线程为什么还能这么快
- Redis 6 之后多线程是什么角色

### 问题 2：缓存穿透、击穿、雪崩分别是什么

**回答要点**

- 穿透是请求根本不存在的数据，缓存和数据库都被打
- 击穿是热点 key 在失效瞬间被高并发请求穿透
- 雪崩是大量 key 同时过期，导致数据库压力骤增
- 常见治理手段包括布隆过滤器、互斥锁、随机过期时间和降级兜底

**常见追问**

- 热点 key 击穿时为什么不能只靠重试
- 随机过期时间为什么能缓解雪崩

### 问题 3：分布式锁要注意什么

**回答要点**

- 要考虑超时、误删、续期和可用性问题
- 加锁和释放锁需要保证原子性
- 不能只关注“能锁住”，还要关注异常场景下怎么释放和兜底

**常见追问**

- 为什么释放锁通常要校验持有者
- Redlock 争议点是什么
