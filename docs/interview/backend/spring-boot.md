# Spring Boot 面试题

## 高频问题

- 自动配置原理
- Bean 生命周期
- Spring MVC 请求流程
- 常用注解和依赖注入方式

## 常见追问

- Spring Boot 为什么能做到开箱即用
- `@Autowired` 底层依赖什么机制
- 拦截器、过滤器、AOP 分别适合什么场景

## 标准回答框架

1. 先说 Spring Boot 在解决什么问题
2. 再说自动配置或容器机制
3. 最后补工程实践场景

## 题库示例

### 问题 1：Spring Boot 为什么能做到开箱即用

**回答要点**

- 核心在于自动配置机制
- 它会根据类路径、配置文件和条件注解自动装配常见组件
- 开发者不用从零手写大量 XML 或基础配置

**常见追问**

- 自动配置是怎么生效的
- 哪些场景下你会覆盖默认配置

### 问题 2：Bean 生命周期大致是什么

**回答要点**

- Bean 会经历实例化、依赖注入、初始化、使用和销毁等阶段
- Spring 允许通过初始化回调和后置处理器扩展这些阶段
- 理解这个过程有助于分析依赖注入和 AOP 行为

**常见追问**

- `BeanPostProcessor` 在哪里起作用
- `@PostConstruct` 和初始化方法有什么关系

### 问题 3：拦截器、过滤器、AOP 怎么区分

**回答要点**

- 过滤器更接近 Servlet 规范层，适合处理通用请求入口逻辑
- 拦截器更偏 MVC 层，适合权限、日志、请求校验等
- AOP 更适合抽离方法级横切逻辑

**常见追问**

- 登录鉴权更适合放哪一层
- 为什么有些日志逻辑更适合 AOP
