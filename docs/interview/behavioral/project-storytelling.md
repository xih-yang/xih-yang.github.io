# 项目介绍模板

## 推荐结构

1. 项目背景和目标
2. 你的角色和负责范围
3. 核心技术方案
4. 关键难点与取舍
5. 结果和复盘

## 常见追问

- 为什么这个难点是你来负责
- 方案有没有备选
- 最终结果怎么衡量

## 易错点

- 只讲项目背景，不讲你自己的动作
- 结果没有量化，听起来像流水账

## 题库示例

### 问题 1：请介绍一个你做过的项目

**回答要点**

- 先说项目背景和目标
- 再说你负责的模块和关键动作
- 再讲一个具体技术难点和你的取舍
- 最后用结果或指标收尾

**常见追问**

- 你在里面的角色到底是什么
- 如果重来一次你会怎么优化
