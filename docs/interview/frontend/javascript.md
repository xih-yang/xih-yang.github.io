# JavaScript 面试题

## 高频问题

- 作用域、闭包、执行上下文
- 原型链和继承
- `this` 绑定规则
- Promise、事件循环、宏任务与微任务
- 防抖与节流
- 深拷贝和浅拷贝

## 准备建议

- 这类题要能画图解释
- 高频手写题要会写，也要会讲边界条件

## 常见追问

- 闭包为什么不会被回收
- `new` 关键字做了什么
- Promise 链里异常是怎么传递的
- 事件循环里 `setTimeout` 和 `Promise` 为什么顺序不同

## 标准回答框架

1. 先说机制本质
2. 再讲执行过程
3. 再给代码例子
4. 最后讲常见坑

## 易错点

- 只背答案，不会推导执行顺序
- 能写手写题，但说不出边界条件

## 题库示例

### 问题 1：什么是闭包

**回答要点**

- 闭包本质上是函数和其词法作用域环境的组合
- 即使外层函数执行结束，内部函数仍然可以访问外层变量
- 常见场景包括缓存、私有变量、函数工厂和事件处理

**常见追问**

- 为什么闭包里的变量不会立刻被回收
- 闭包会不会引起内存泄漏

### 问题 2：事件循环是什么

**回答要点**

- JavaScript 主线程一次只执行一个任务
- 同步代码先执行，异步任务会进入任务队列
- 微任务通常会在当前宏任务结束后优先执行
- Promise 回调和 `MutationObserver` 属于微任务，`setTimeout` 属于宏任务

**常见追问**

- 为什么 `Promise.then` 一般先于 `setTimeout`
- `async/await` 和 Promise 的关系是什么

### 问题 3：`this` 是怎么绑定的

**回答要点**

- `this` 取决于调用方式，而不是定义位置
- 常见绑定规则包括默认绑定、隐式绑定、显式绑定和 `new` 绑定
- 箭头函数没有自己的 `this`，会继承外层作用域

**常见追问**

- `call`、`apply`、`bind` 有什么区别
- 箭头函数为什么不能当构造函数
