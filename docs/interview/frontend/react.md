# React 面试题

## 高频问题

- React 为什么需要 key
- Hooks 为什么不能放在条件判断里
- `useEffect` 的执行时机
- 受控组件与非受控组件
- 状态管理方案如何选型
- React 渲染性能怎么优化

## 项目题方向

- 组件边界怎么拆
- 副作用怎么治理
- 复杂页面状态怎么管理

## 常见追问

- 为什么不要用索引做 key
- `useEffect` 依赖数组缺了会怎样
- 为什么状态更新不是立刻同步到变量上

## 标准回答框架

1. 先说 React 的渲染和状态模型
2. 再讲具体 API 的设计意图
3. 最后结合真实组件场景回答

## 易错点

- 把 `useEffect` 当成生命周期替代品死记
- 讲性能优化时只会背 `memo`，不会分析重渲染来源

## 题库示例

### 问题 1：React 为什么需要 key

**回答要点**

- key 用来帮助 React 在 diff 时识别节点身份
- 稳定 key 可以减少错误复用和不必要的重渲染
- 如果列表会增删改排序，索引 key 很容易引发状态错位

**常见追问**

- 为什么索引 key 在静态列表里问题不大
- key 应该写在什么层级

### 问题 2：`useEffect` 的执行时机是什么

**回答要点**

- 组件渲染提交到页面之后，React 才会执行 effect
- 默认情况下首次渲染后执行，依赖变化后再次执行
- 返回函数会在下一次 effect 执行前或组件卸载时清理

**常见追问**

- 为什么依赖数组缺失会有问题
- `useEffect` 和 `useLayoutEffect` 区别是什么

### 问题 3：React 性能优化一般怎么做

**回答要点**

- 先定位重渲染来源，而不是上来就加缓存
- 常见做法包括组件拆分、减少无效状态提升、稳定 key、按需渲染、虚拟列表
- 工具层面可以用 Profiler、性能面板和埋点数据辅助分析

**常见追问**

- `memo` 是不是加了就一定更快
- 大列表除了虚拟滚动还能怎么优化
