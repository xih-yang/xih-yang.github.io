## 推荐准备顺序

1. 先准备基础知识和高频八股。
2. 再整理项目经历，形成稳定表达。
3. 然后补算法和系统设计题。
4. 最后准备行为面试和反问问题。

## 面试通用技巧

- 回答时尽量按“结论 -> 原理 -> 场景 -> 风险”来组织。
- 项目题优先讲你亲手做过、能展开细节的部分。
- 遇到不会的问题，先给分析路径，不要直接沉默。

## 通用回答框架

1. 先给结论
2. 再解释原理
3. 补一个真实场景
4. 说明边界或常见误区

## 项目题准备建议

- 每个项目至少准备一个“技术难点”
- 每个难点都准备“为什么这样选”和“如果重来会怎么做”
- 结果最好有可量化指标

## 使用建议

- 基础知识题和项目题最好交替准备
- 每看完一页，尝试口头复述一次，比只读更有效
