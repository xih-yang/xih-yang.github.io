# 高并发与缓存

## 高频问题

- 限流、降级、熔断怎么理解
- 缓存和数据库一致性如何处理
- 热点数据如何抗住高并发

## 常见追问

- 为什么单纯加机器不一定解决问题
- 热点 key 怎么发现和治理
- 一致性和性能冲突时怎么取舍

## 标准回答框架

1. 先识别瓶颈在哪一层
2. 再给缓存、削峰、扩容等方案
3. 最后说明代价和取舍

## 题库示例

### 问题 1：高并发系统为什么不能只靠加机器

**回答要点**

- 如果瓶颈在数据库锁、热点 key 或单点依赖，加机器不一定有效
- 有些问题是架构层和数据层瓶颈，不是应用层算力不足
- 所以通常要先定位瓶颈层，再决定扩容、缓存还是削峰

**常见追问**

- 热点请求一般怎么识别
- 限流和降级通常放在哪一层
