# 微服务与接口治理

## 高频问题

- 服务拆分依据是什么
- 接口网关和鉴权怎么做
- 如何做链路追踪和问题排查

## 常见追问

- 服务拆得太细会有什么问题
- 网关和业务服务边界怎么划
- 线上链路追踪通常需要哪些关键信息

## 标准回答框架

1. 先说为什么要治理
2. 再说拆分、网关、追踪分别解决什么问题
3. 最后讲落地时的协作成本

## 题库示例

### 问题 1：服务拆分依据是什么

**回答要点**

- 优先按业务边界和团队协作边界拆，而不是按技术喜好乱拆
- 拆分后要考虑接口稳定性、数据归属和调用链复杂度
- 拆得太细会带来治理和运维成本

**常见追问**

- 什么时候不应该继续拆
- 拆分之后跨服务事务怎么办
