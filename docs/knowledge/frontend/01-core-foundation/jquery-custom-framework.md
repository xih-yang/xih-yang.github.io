# jQuery 与自定义框架

jQuery 是理解“早期前端如何提升开发效率”的重要切口，自定义框架封装则能帮助理解现代框架到底抽象了什么。

## 核心范围

- jQuery 选择器、事件、动画、AJAX、插件机制
- 常见后台管理系统的 jQuery 实战模式
- 原生框架封装：事件总线、模板拼接、组件雏形、工具函数库
- 封装分层：基础工具层、中级组件层、高级业务框架层

## 学习价值

- 帮助理解链式调用、插件化、DOM 批处理等思路
- 有助于从“会用框架”过渡到“知道框架为什么这样设计”

## 常见问题

- 把 jQuery 当成过时技术直接跳过，错过很多框架抽象基础
- 封装时只有工具函数，没有形成模块边界和职责分层

## 学习路线

1. 先理解 jQuery 如何简化 DOM、事件和 AJAX。
2. 再分析插件化和链式调用背后的抽象思路。
3. 最后尝试自己封装一套轻量工具层或组件层。
