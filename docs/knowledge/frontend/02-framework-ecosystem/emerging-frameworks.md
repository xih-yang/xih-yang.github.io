# 新兴高性能框架

这些框架通常围绕更细粒度的响应式、更少的运行时开销或更快的首屏恢复能力展开。

## 关注对象

- Svelte：编译时框架，减少运行时抽象
- SolidJS：细粒度响应式，接近原生性能
- Qwik：可恢复性优先，面向极致首屏与懒执行
- Lit：Web Components 友好，适合设计系统和组件库
- Preact：轻量 React 兼容方案
- Vanilla：无框架开发，适合理解本质与小体量应用

## 学习建议

- 对比它们在响应式模型、编译产物、心智负担和生态成熟度上的差异

## 观察维度

- 是编译时优化，还是运行时优化
- 是细粒度响应式，还是组件级更新
- 生态是否成熟，是否适合生产落地

## 学习路线

1. 先理解这些框架想解决的 React / Vue 痛点。
2. 再对比它们的响应式和渲染模型。
3. 最后选一两个做小项目实验，不建议一开始就全量投入生产。
