# 自动化构建与测试

自动化是把重复工作标准化，测试则是把质量验证前移。

## 核心范围

- 自动化构建：Grunt、Gulp 的历史定位与任务编排
- 单元测试：Jest、Vitest 的断言、Mock、覆盖率与组件测试
- E2E 测试：Cypress、Playwright 的页面流程验证
- 测试分层：单元、集成、端到端的职责边界

## 实战关注点

- 测试不要只追求覆盖率数字
- 优先保证核心业务链路和容易回归的风险点

## 常见问题

- 单元测试写了很多，但关键流程没人兜底
- E2E 用例太重，执行慢且维护成本高
- 自动化任务很多，但和发布链路没有打通

## 学习路线

1. 先建立单元、集成、E2E 的基本分层认知。
2. 再用 Jest / Vitest 和 Playwright / Cypress 落地。
3. 最后把自动化测试接入 CI 和发布流程。
