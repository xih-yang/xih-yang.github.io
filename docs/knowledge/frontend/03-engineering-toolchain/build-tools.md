# 构建与编译工具

构建工具主要处理模块打包、开发服务、资源优化和代码转换。

## 核心范围

- 打包工具：Webpack、Rollup、Parcel、esbuild
- 新一代开发工具：Vite、VitePlus 及其插件机制
- 编译转换：Babel、SWC 的语法降级、语法提案支持和性能差异
- 常见能力：代码分割、Tree Shaking、按需加载、环境变量、多环境构建

## 选型关注点

- 开发启动速度
- 生产构建性能
- 插件生态成熟度
- 团队已有技术栈兼容性

## 常见问题

- 只看打包速度，不看插件生态和团队可维护性
- 配置堆很多，但并不理解构建链路每一步在做什么
- 把 Babel、SWC、打包器和运行时职责混在一起

## 学习路线

1. 先理解模块打包、开发服务器和代码转换的区别。
2. 再学习 Webpack、Vite、Rollup 各自适合什么场景。
3. 最后补代码分割、Tree Shaking 和多环境构建。
