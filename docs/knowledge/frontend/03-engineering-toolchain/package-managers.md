# 包管理工具

包管理工具决定了依赖安装速度、锁文件策略、工作区管理方式和团队一致性。

## 核心范围

- npm：默认生态入口、脚本管理、发布流程
- pnpm：硬链接与去重策略、Monorepo 友好性
- Yarn：经典版与 Berry 的差异
- Bun：一体化运行时、包管理和脚本能力
- Bower：历史工具，了解其定位即可

## 实战关注点

- 锁文件统一
- 私仓配置与镜像源管理
- Monorepo 工作区和依赖边界

## 常见问题

- 团队里混用多种包管理工具，导致锁文件混乱
- 没有理解 hoist、去重和硬链接策略
- Monorepo 下工作区依赖边界不清晰

## 学习路线

1. 先掌握依赖安装、脚本执行和锁文件作用。
2. 再对比 npm、pnpm、Yarn、Bun 的差异。
3. 最后把包管理放进 Monorepo 和 CI 场景里理解。
