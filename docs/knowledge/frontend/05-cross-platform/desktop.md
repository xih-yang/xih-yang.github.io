# 桌面端开发

桌面端方案适合需要文件系统、系统托盘、硬件访问或离线能力更强的业务。

## 核心范围

- Electron：Chromium + Node.js 的能力组合、主进程与渲染进程
- Tauri：Rust 后端 + Web 前端，体积更小、资源占用更轻
- 桌面端常见需求：自动更新、安装包分发、本地存储、系统权限

## 实战关注点

- 主进程、渲染进程和系统权限边界要清晰
- 自动更新和签名分发通常比页面开发更容易踩坑
- 桌面应用也要关注内存占用和启动速度
