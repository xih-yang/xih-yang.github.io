# 混合 App

混合 App 关注的是 JavaScript 与原生能力之间的桥接成本、性能上限和交付效率。

## 核心范围

- React Native：组件模型、桥接、导航、状态管理与原生模块接入
- Expo：开发体验、构建分发和能力边界
- Flutter：Dart、渲染引擎、跨平台一致性和前端协作方式

## 选型关注点

- 是否需要高性能原生交互
- 是否需要频繁调用原生能力
- 团队是否能承接多语言栈

## 常见问题

- 只看跨端效率，不评估性能和原生能力差距
- 桥接层设计混乱，JS 和原生边界不清晰
- 构建、调试和发版流程没有提前打通
