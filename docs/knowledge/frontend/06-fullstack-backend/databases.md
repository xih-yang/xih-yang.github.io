# 数据库体系

前端不一定要成为 DBA，但需要理解不同数据库如何服务于接口和业务场景。

## 核心范围

- 关系型数据库：MySQL 的表结构、索引、事务和常见查询
- 非关系型数据库：MongoDB 的文档模型、Redis 的缓存与会话能力
- 向量数据库：面向 AI 检索场景的数据存储与召回基础

## 学习重点

- 理解数据建模和接口返回结构的关系
- 理解缓存、持久化和一致性之间的取舍

## 常见问题

- 把数据库只当成“后端的事”，导致接口设计理解不到位
- 不清楚关系型和非关系型数据库各自适合什么业务
- 看到 Redis 就只想到缓存，没有想到分布式协作场景
