# Node 全栈与 BFF

Node.js 让前端可以更自然地承接接口聚合、SSR 服务、鉴权中间层和全栈项目。

## 核心范围

- Node.js 基础：事件循环、模块系统、文件系统、HTTP 服务
- Express：中间件模型、路由组织、接口封装
- NestJS：模块化、依赖注入、装饰器和企业级组织方式
- Koa：洋葱模型、中间件与轻量扩展
- BFF 层：接口聚合、权限控制、适配多端和防止前端直连复杂后端

## 实战关注点

- 前端做 Node 服务时，重点先放接口聚合和中间层治理
- 框架选型要看团队规模和规范要求
- SSR、鉴权和 BFF 通常是最容易落地的 Node 场景
