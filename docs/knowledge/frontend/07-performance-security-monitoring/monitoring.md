# 监控体系

监控体系的目标不是“收集很多数据”，而是出现问题时能快速知道影响范围、复现路径和优先级。

## 核心范围

- 错误上报：JS 异常、Promise 异常、资源加载失败、接口异常
- 性能监控：Web Vitals、页面耗时、接口耗时、长任务
- 用户行为埋点：曝光、点击、转化漏斗、行为路径
- 平台建设：告警、聚合、采样、版本关联、Source Map 管理

## 实战关注点

- 监控指标要能对上业务问题和用户反馈
- 告警不要泛滥，重点是能真正推动排障
- 版本、环境、用户行为和错误日志需要能串起来

## 常见问题

- 指标收集很多，但没有人用它来定位问题
- Source Map、版本号和发布记录没有打通
- 埋点口径频繁变化，导致数据无法长期对比
