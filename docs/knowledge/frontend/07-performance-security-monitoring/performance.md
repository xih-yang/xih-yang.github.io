# 性能优化

性能优化要区分加载性能、运行时性能和感知性能，不能只盯着某一个指标。

## 核心范围

- 加载优化：资源压缩、缓存、懒加载、预加载、代码分割
- 运行时优化：减少重渲染、虚拟列表、长任务拆分、主线程治理
- 指标体系：FCP、LCP、CLS、INP、TTFB
- 工具：Lighthouse、Performance 面板、Web Vitals

## 实战关注点

- 优化前先测量，优化后再验证
- 把页面加载、交互响应和运行时卡顿分开分析
- 性能治理最好进入监控平台，而不是只做一次性优化

## 常见问题

- 把所有性能问题都归结为包太大
- 只看 Lighthouse 分数，不看真实用户体验
- 优化项很多，但没有优先级和收益评估
