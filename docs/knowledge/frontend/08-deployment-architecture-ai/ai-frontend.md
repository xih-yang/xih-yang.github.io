# AI 前端全体系

AI 前端关注的不是“调用一次模型接口”，而是把模型能力稳定地接进产品体验、数据链路和性能约束里。

## 核心范围

- Web AI 框架：TensorFlow.js、浏览器端推理与端侧能力边界
- 大模型对接：多轮对话、函数调用、工具编排、模型网关
- 流式输出：SSE / WebSocket 传输、打字机效果、取消与重试
- RAG 应用：检索、召回、重排、上下文拼接和结果展示
- AIGC 实战：文案生成、图像生成、表单助手、智能问答
- AI 组件与性能优化：消息流组件、虚拟滚动、Token 级更新、成本控制

## 实战关注点

- 模型能力、产品体验、延迟和成本要一起平衡
- 流式输出和聊天 UI 是 AI 前端的基础设施
- RAG 不只是检索，还包括上下文组织和答案呈现

## 常见问题

- 把 AI 接入理解成单次接口调用，忽略完整交互链路
- 没有处理流式中断、重试、上下文截断和错误兜底
- 只关注模型效果，不关注浏览器性能和 Token 成本
