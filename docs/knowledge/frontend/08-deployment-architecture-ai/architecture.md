# 高级架构

架构话题的重点是解决复杂团队协作、大规模项目拆分和复用效率问题。

## 核心范围

- 微前端：应用拆分、运行时集成、团队边界和通信
- Monorepo：工作区、共享包、版本策略、构建缓存
- 模块联邦：远程模块加载、共享依赖、发布协同
- 可视化 / 3D：数据可视化、Canvas、WebGL、Three.js 基础方向

## 实战关注点

- 架构升级要有明确问题驱动，别为了“高级”而上复杂方案
- 团队边界、发布节奏和依赖共享往往比技术细节更关键
- 可视化和 3D 场景要重点关注性能与交互设计

## 常见问题

- 项目还不复杂，就提前引入微前端或模块联邦
- Monorepo 只解决了代码放一起，没有解决协作流程
- 架构图很好看，但没有带来真实的效率收益
