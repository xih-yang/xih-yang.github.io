# 容器化部署与 CI/CD

部署能力决定了项目能否稳定上线、快速回滚和标准化交付。

## 核心范围

- Docker：镜像、容器、构建、多阶段构建
- Nginx：静态资源托管、反向代理、缓存、压缩与路由回退
- CI / CD：自动测试、自动构建、自动发布、灰度与回滚
- 常见平台：GitHub Actions、GitLab CI、Jenkins、云平台流水线

## 实战关注点

- 构建产物、环境变量和发布流程要标准化
- 灰度、回滚和缓存刷新要提前演练
- 前端部署不是“把 dist 扔上去”，还要管代理和路由

## 常见问题

- 本地能跑，线上发布链路却缺少标准化配置
- Nginx 回退、缓存和静态资源策略配错
- CI 只做构建，不做质量门禁和回滚准备
