# 工具文档与协作规范

工具协作能力决定了团队能否顺畅配合，也决定了知识能否沉淀下来。

## 核心范围

- 版本控制：Git / SVN 的基本能力与协作流程
- 表达工具：流程图、思维导图、原型图、架构图
- 接口文档：OpenAPI、Swagger、Apifox、协作约定
- 国际化：i18n、文案管理、语言包拆分与时区处理

## 实战关注点

- 文档不是补材料，而是协作的一部分
- 流程图、接口文档和国际化规范要跟项目一起演进
- 工具选型应优先考虑团队共享效率

## 常见问题

- 有代码没文档，新人和跨团队协作成本很高
- 接口文档和真实接口脱节
- i18n 只做文字替换，忽略日期、时区和格式化规则
